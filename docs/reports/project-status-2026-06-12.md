## 一、项目目标

项目用于自动完成腾讯云云防火墙告警的增量采集、模型研判、安全处置和企微通知。

核心目标：

- 每小时处理新增告警，避免反复分析历史数据。
- 结合告警字段和可用源包证据，判断攻击是否成功。
- 自动忽略明确扫描、明确失败和未见成功证据的告警。
- 保留确认成功、高危和证据不足的告警供进一步复核。
- 排除腾讯云暴露面扫描 IP 和公司漏扫 IP。
- 不自动封禁任何 IP。

## 三、整体处理流程

### 1. 每小时采集

`run_collect.ps1` 调用：

```powershell
python cfw_alert_monitor.py collect --lookback-hours 1
```

处理步骤：

1. 拉取上一小时全流量检测与响应日志。
2. 拉取未处理告警摘要。
3. 排除腾讯云扫描源和公司漏扫源。
4. 对新增事件去重并写入本地数据目录。
5. 压缩字段后调用模型进行批量研判。
6. 对需要复核且存在源包摘要的告警进行第二次源包复核。
7. 对允许自动处理的告警执行忽略。
8. 对告警中心最近两小时数据执行安全规则处置。
9. 将处理摘要发送至企微。

告警中心使用两小时窗口是为了覆盖云端聚合和入库延迟；原始流量分析仍严格使用上一小时窗口。

### 2. 每日报告

`run_daily_report.ps1` 执行最近两天未处理告警的补充研判，然后生成当天汇总报告并发送企微。

日报包括：

- 唯一告警数量。
- 攻击 IP 数量。
- 告警等级分布。
- 模型研判结果分布。
- 自动忽略和处置失败数量。
- 主要攻击 IP 和主要事件。

### 3. 模型研判方式

项目不是 ReAct 工具循环，也没有使用 OpenAI SDK function call。

当前方式为：

1. 从本机 `~/.codex/auth.json` 读取 Codex 登录鉴权。
2. 直接请求 Codex Responses 接口。
3. 要求模型返回结构化 JSON。
4. Python 代码解析结果并决定是否调用腾讯云处置接口。

腾讯云查询、加白和忽略操作均由本地 Python 代码执行，模型不能直接操作云资源。

## 四、研判和自动处置规则

允许自动忽略的结果：

- `扫描探测`
- `确认未成功`
- `未见成功证据`

必须保留的结果：

- `确认成功`
- `需人工复核`
- 高危且缺少充分证据的告警

判断攻击成功需要至少存在一种有效成功证据：

- 命令执行回显。
- 文件成功写入。
- WebShell 行为。
- 敏感数据有效返回。
- 回连或其他明确落地证据。

单独出现攻击请求、HTTP 200、普通页面、ETag 或哈希样字符串不能作为攻击成功证据。

## 五、安全边界

- 不自动封禁攻击 IP。
- 腾讯云扫描 IP 和公司漏扫 IP 不执行封禁。
- 真实 `config.json`、企微 webhook、腾讯云凭证、Codex 鉴权、生产告警、源包和日志不进入 Git 仓库。
- 模型只输出研判结果，云端处置由代码中的固定规则控制。
- 模型不可用时使用本地规则降级，不会把证据不足告警自动判为成功或直接忽略。

## 六、模型降级说明

当前“模型降级”不是模型被切换，也不是从 `gpt-5.5` 降到其他模型。

日志中观察到的直接原因：

- `WinError 10060`：连接 Codex Responses 接口超时。
- `WinError 10061`：目标连接被拒绝。

发生连接异常时：

1. 错误写入 `logs/llm-errors.jsonl`。
2. 主流程继续执行。
3. 对无法获得模型结果的告警使用本地规则兜底。
4. 证据不足的结果保留复核，不自动忽略。

因此模型降级会增加保留告警数量，但不会导致高风险告警被自动忽略。

## 七、当前已知问题

| 优先级 | 问题 | 影响 | 建议 |
|---|---|---|---|
| 高 | Codex Responses 接口偶发超时或拒绝连接 | 模型降级，人工复核数量增加 | 增加短超时重试、连接预检和失败批次重试队列 |
| 中 | 告警中心部分事件没有可关联源包 | 无法确认攻击成功或失败 | 增加源包补拉和资产侧日志关联 |
| 中 | PowerShell 日志中部分中文乱码 | 影响日志阅读，不影响云端处置 | 统一子进程 stdout/stderr 为 UTF-8 |
| 中 | 高危扫描类事件仍默认保留 | 噪声偏高 | 为明确内部扫描资产建立受控白名单 |
| 低 | 当前缺少自动化单元测试 | 修改后依赖人工验证 | 为研判归一化和处置边界增加测试 |
